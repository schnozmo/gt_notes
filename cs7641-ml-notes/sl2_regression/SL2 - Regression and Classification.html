fit f(x) = C0 + C1X1 + C2X**2 + C3X**3 + CkX**k
k = 0 => constant
k = 1 => line
k = 2 => quadratic

as k approaches data points count, error decreases, overfit risk increases

Errors - Training data has error not modeling f(x), but f(x) + e
- sensor error
- malicious - not being truthful with poll data
- transcription - clerical error
- unmodeled influence -  too simple of a model that doesn’t incorporate key features

Cross-Validation
- model complex enough that it fits the model in training but general enough to work on the test data
- cross validation set is subset of training used as an initial test
- split training set in to folds
    if 4 folds
        run1 - train on 1,2,3 test on 4
        run2 - train on 2,3,4 test on 1
        run3 - train on 3,4,1 test on 2
        run4 - train on 4,1,2 test on 3
        take average of outputs
- cross-validation error drops as power increases from constant, line, quaratic, etc, but increases when it begins to overfit on some folds, wihle traiing error still falls
