Curse of Dimensionality - amount of data needed grows 2^N which number of features


Feature selection is an exponential problem. It’s an optimization problem over an arbitrary set of variables.


Filtering - set of features as input. Some algorithm that maximizes some criteria. It outputs some subset of features to a learner
+ speed
- features looked at in isolation from each othe
- no feedback, ignores the learning problem
Wrapping - set of features as input. Iterate of some combination of features presented to a learner and evaluates the output of the learner
+ takes into account model bias and learning 
- very slow

Search (uses domain knowledge)
- Information gain used as a measure of feature selection. How much information does a feature give me about a class label
- variance, entropy (GINI index) - features that show up a lot
- remove dependent, redundant features. 

Search could be a decision tree and passes on the set of features used by the decision tree to the real learner

Wrapping
- hill climbing
- randomized optimization
- forward search
    -> start with a feature, look at all in isolation, keep best feature
    -> go through remaining features, paired with best, keep 2nd best
    -> until error is not significantly better
- backward search
    -> start with all feature, remove one in isolation, evaluate combination of rest, eliminate worst feature
    -> go through remaining features, removing one in isolation, evaluate combination of rest, eliminate next worst feature
    -> until error not significantly better

Relevance - measures effect on the Bayes Optimal Classifier
- x_i is strongly relevant if removing it degrades the Bayes Optimal Classifier
- x_i can still be weakly relevant if there are a subset of features S s.t. adding x_i to S improves the Bayes Optimal Classifier
- x_i is irrelevant otherwise

Usefulness - measures effect on a particular predictor, effect on error given a model or learner
